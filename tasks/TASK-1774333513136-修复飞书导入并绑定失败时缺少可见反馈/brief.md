# 修复飞书导入并绑定失败时缺少可见反馈

## 目标

- 修复 Feishu 中点击“导入并绑定”后，当 `threadId` 不存在、导入失败或绑定失败时，用户侧看起来像“没有反应 / 卡住”的问题。

## 范围

### In

- 导入表单提交后的失败反馈可见性
- 区分常见失败类型：空 `threadId`、不存在的 `threadId`、绑定冲突
- 当前导入表单卡与补充通知的交互策略

### Out

- imported thread 本身的 runtime 恢复逻辑
- 非导入链路的普通 Feishu 消息回执
- 桌面端或 VSCode 侧交互

## 验收标准

- 用户提交无效 `threadId` 后，Feishu 上能明确看到失败原因，而不是只感觉“没反应”
- 当前表单仍能保留或回显已提交的 `threadId`，便于继续修改
- 长连接 / 移动端缺失 `open_message_id` 的情况下，失败反馈仍然能稳定显示
